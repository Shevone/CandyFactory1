Лаба 2

От нас требуется добавить в наш проект так называемую кастомную коллекцию для нашей иерархии классов, что это?
Это какой то класс класс, который будет содердать в себе несколько объектов нашей иерархии. 

Как класс сделать коллекцией, необходимо добавить внутрь какой то объект который будет хранить информацию(например список объектов)
А так же добавить реализацию какого либо интерфеса коллекции, чтобы наша коллекция реализовывала необходимые методы методы.

Для начала мы определили интерфес ICandyCollection - Это некий шаблон того, как должна выглядить наша кастомная коллекци, для наглядности добавили 2 метода,
которые считают суммарную себистомиость и суммарную рыночную стоимость всех конфет в коллекции
А так же мы определили что каждый класс, реалзиующий интерфес ICandyCollection будет реализовывать интерфес ICollection, вот так:
public interface ICandyCollection<T> : ICollection<T>

Мы выбрали интерфес ICollection, тк он содержит самый крутой миниму
-Add
-Remove
-Contains
-CopyTo
-Clear
-GetEnumerato (это важный метод, он возрващает нам объект для итераиции например если мы захотим отправить нашу коллекцию в цикл foreach)
- свойство count

Определяем что наш класс будет реализовывать наш кастомный интерфеус
public class CandyCollection<T> : ICandyCollection<T> where T: Candy
Наш класс коллекция является обобщенным, то есть мы можем использовать его под разные типы, но мы определям что эти типы это
либо класс Candy, либо его наследник (вот этим выраженим where T: Candy)

в самой програме работаем так. Создали класс Конфетной фабрики. Где определил список конфет по 3м разным видам. Взаимодейсвтие происходит именно через интерфейс
Пример вызовова метода интерфейса находится в методе добавления новой конфеты. Там мы вызываем метод add.
